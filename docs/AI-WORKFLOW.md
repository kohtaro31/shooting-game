### 作業中のルール

- 古いメモや雰囲気から要件を"補完"しない。
- 仮定は明示してから進める（「X/Y/Zを仮定しています。正しいですか？」）。
- 変更は小さく・可逆・レビュー可能に保つ。
- 実装後は「何が変わったか」と文書更新案（差分）を提示し、ユーザーのレビューを求める。
- 曖昧なら止まって質問する。

## 5つのゲート（Gate A〜E）

| Gate | 内容 |
|---|---|
| A | 質問先行。不足している制約を確認し、仮定を明示する。 |
| B | 小さなストーリー単位（15〜30分で検証可能なサイズ）で進める。 |
| C | 非同期ループ。文書がターン間の共有メモリ。 |
| D | 検証後に差分報告＋文書更新案を提示。ユーザーがapprove/edit/reject。 |
| E | コミットは人間が主導。AIは変更をマイクロステップで整える。 |
